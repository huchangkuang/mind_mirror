## Context

Mind Mirror 是一个基于 Next.js 15 + TypeScript + Tailwind CSS 的心理测试平台。目前仅实现了 MBTI 测试，使用 Zustand 进行状态管理，sessionStorage 保存测试进度，localStorage 保存历史记录。UI 采用毛玻璃效果（glassmorphism）风格，使用 Outfit + Work Sans 字体组合。

现有架构：
- `app/mbti/` - MBTI 测试页面（介绍、测试、结果、历史）
- `app/page.tsx` - 当前为 MBTI 宣传首页
- `components/ui/` - 通用 UI 组件（Button, Card, ProgressBar）
- `stores/mbti-store.ts` - MBTI 状态管理
- `lib/mbti/` - MBTI 业务逻辑

## Goals / Non-Goals

**Goals:**
- 创建测试导航首页，展示所有可用测试项目
- 实现完整的性格匹配城市测试功能
- 保持与现有 MBTI 测试一致的设计语言和用户体验
- 复用现有 UI 组件和样式系统

**Non-Goals:**
- 不重构 MBTI 测试的现有功能
- 不添加用户系统或后端数据库
- 不实现测试之间的数据关联

## Decisions

### 1. 首页设计模式：卡片式导航
- **决策**: 采用卡片网格布局展示测试项目
- **理由**: 
  - 视觉上与现有毛玻璃设计风格一致
  - 每个卡片包含：图标、标题、简介、预计时长、开始按钮
  - 易于扩展新测试项目
  - 用户一目了然所有可用选项
- **替代方案**: Tab 切换式 - 被拒绝，因为不够直观且难以展示测试详情

### 2. 性格匹配城市测试的评分维度
- **决策**: 采用 4 维度评分模型（生活方式、社交偏好、环境偏好、节奏偏好）
- **理由**:
  - 与 MBTI 的 4 维度模型保持一致，降低用户认知负担
  - 每个维度对应不同的城市类型特征
  - 便于生成多维度的结果分析
- **维度定义**:
  - 生活方式: 传统保守 ←→ 现代开放
  - 社交偏好: 独处内向 ←→ 社交外向
  - 环境偏好: 自然宁静 ←→ 都市繁华
  - 节奏偏好: 慢节奏 ←→ 快节奏

### 3. 城市匹配算法
- **决策**: 基于维度得分区间匹配 3-5 个推荐城市
- **理由**:
  - 避免单一结果带来的"非此即彼"感
  - 提供多元选择，增加趣味性
  - 每个城市附带匹配度百分比
- **匹配逻辑**: 
  - 计算用户在各维度的得分（-100 到 +100）
  - 与预设城市特征向量计算余弦相似度
  - 返回相似度最高的 3 个城市

### 4. 数据结构设计
- **决策**: 复用 MBTI 的题库 JSON 格式
- **理由**:
  - 保持项目数据格式一致性
  - 减少学习成本
  - 便于后续统一管理和扩展
- **格式示例**:
```json
{
  "meta": { "version": "1.0", "questionCount": 12, "estimatedMinutes": 5 },
  "questions": [
    {
      "id": "q1",
      "text": "...",
      "options": [
        { "text": "...", "dimensionWeights": { "lifestyle": -10, "social": 5 } }
      ]
    }
  ]
}
```

### 5. 路由结构
- **决策**: 平行路由结构 `/city-match/*` 与 `/mbti/*` 对应
- **理由**:
  - 保持 URL 语义清晰
  - 便于代码组织和维护
  - 未来添加新测试时遵循相同模式

### 6. 状态管理
- **决策**: 为城市测试新建独立的 Zustand store
- **理由**:
  - 避免与 MBTI 状态混淆
  - 支持同时保存两种测试的历史记录
  - 便于独立测试和调试

## Risks / Trade-offs

- **[风险]** 首页从 MBTI 专属变为导航页，可能影响直接访问用户的体验
  - **缓解**: 保持原有视觉风格，MBTI 作为第一个/推荐卡片突出展示
  
- **[风险]** 城市测试的题目设计需要专业性和趣味性平衡
  - **缓解**: 参考 MBTI 的题目风格，保持简洁易懂，每题 2-4 个选项
  
- **[风险]** 城市匹配结果可能不够准确或全面
  - **缓解**: 
    - 结果页显示"仅供参考"的免责声明
    - 提供多个城市选择而非单一答案
    - 允许用户重新测试

- **[权衡]** 两个测试的存储相互独立，无法分析用户跨测试行为
  - **接受**: 当前阶段不实现用户系统，每个测试作为独立功能

## Migration Plan

4. 测试两种测试的完整流程

**回滚策略**:
- 保留原 `app/page.tsx` 的备份，如有问题可快速恢复
